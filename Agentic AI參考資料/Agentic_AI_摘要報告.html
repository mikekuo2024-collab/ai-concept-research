Agentic AI：定義、起源與演進背景

從語言模型到自主行動者的典範轉移——理解 Agentic AI 的本質、發展脈絡，以及它如何從概念走向企業現實

核心定義

Agentic AI 是指能夠感知環境、自主推理、制定計劃並採取行動以達成目標的 AI 系統，具備半自主至全自主的運作能力。與 Generative AI 最根本的差異在於：Agentic AI 是主動的（Proactive），而不是被動回應的（Reactive）。

🧩五大核心特徵
🎯
自主性

自主性（Autonomy）

無需每步驟人類指令，能夠自行做出決策並執行行動，實現長時間的連續運作

🏆
目標導向

目標導向（Goal-Orientation）

接收高層次目標後，自行分解為可執行的子任務序列，持續追蹤直至目標達成

👁️
感知能力

感知（Perception）

⚡

行動（Action）

🔄

學習與適應（Learning）

核心架構與操作流程

Agentic AI 的技術骨架：感知-推理-規劃-行動循環、ReAct 框架、多智能體架構，以及記憶系統設計

♾️Agentic AI 核心執行循環（ReAct 框架）
1

感知（Perceive）

接收使用者目標或環境觸發事件，理解多模態輸入（文字、結構化資料、API 輸出），建立當前世界模型

2

思考（Thought / Reason）

輸出明確的推理過程（Thought），分析當前狀態，確定最佳下一步行動。ReAct 的核心洞見：思考過程是引導行動的路徑圖

3

規劃（Plan）

將高層次目標分解為具體可執行的子任務序列，評估工具選項，安排執行順序與依賴關係

4

行動（Act）

呼叫工具（搜尋、計算器、程式碼執行器、API）、執行子任務、或與其他 agents 溝通

5

觀察（Observe）

接收工具呼叫回傳結果（Observation），評估行動是否達到預期效果，更新世界模型

6

反思與迭代（Reflect & Iterate）

判斷任務是否完成：若是，生成最終輸出；若否，回到步驟 2 繼續規劃，直至達成目標

# ReAct 框架標準執行模式示例
Goal: 「找出 Agentic AI 市場規模並整理成報告」

Thought: 我需要搜尋 Agentic AI 市場規模的資料

Action: WriteFile["agentic_ai_report.md", content=...]
Observation: {success: true}

Action: Finish["報告已完成，涵蓋市場規模與採用率數據"]
🏗️三類多智能體架構
🎖️

垂直架構（Hierarchical）

結構：主 Orchestrator agent 指揮多個 Sub-agents

適用：明確層級任務、分工清晰的工作流

優點：高可控性、易於監督
缺點：Orchestrator 是單點瓶頸

🤝

水平架構（Peer-to-Peer）

結構：多個 agents 平行協作，相互通訊

適用：分散式任務、創意協作、辯論驗證

優點：彈性高、無單點失敗
缺點：協調複雜度高

🔀

混合架構（Hybrid）

結構：結合垂直與水平元素

適用：大規模複雜系統、企業級部署

優點：最靈活、最強大
缺點：最複雜、難以除錯

⚡

Context Window（上下文視窗）：當前對話或任務的即時記憶，隨會話結束消失

💾

長期記憶

向量資料庫（Vector DB）：跨會話持久化儲存，透過語意搜尋檢索相關記憶

🗺️

語意記憶

知識圖譜（Knowledge Graph）：結構化事實與概念關係，用於複雜推理

🛠️

程序記憶

工作流程與技能：已學習的操作流程、工具使用模式、成功解題策略

🔧五種核心 Workflow 模式（Anthropic）
A

Prompt Chaining（提示鏈）

串聯多個 LLM 呼叫，每步驟輸出成為下步驟輸入。適合：文件撰寫 → 審查 → 潤稿的多階段任務

B

Routing（路由分發）

分類輸入，路由到不同處理流程或專業 sub-agent。適合：客服問題分類後轉到對應部門 agent

C

Parallelization（平行化）

同時執行多個獨立子任務，最後彙整結果。適合：同時搜尋多個資訊來源再整合

D

Orchestrator-Subagents（主從協調）

主 agent 負責規劃與協調，sub-agents 各自負責專業任務。適合：軟體開發中的需求分析/編碼/測試分工

E

Evaluator-Optimizer（評估優化）

一個 agent 生成初稿，另一個 agent 評估並提出改善要求，反覆迭代直至品質達標

實戰指南、風險管理與資源

企業導入 Agentic AI 的成熟度路徑、四大核心風險、最佳實踐清單，以及完整參考來源

📈企業採用成熟度模型（五個階段）
成熟度描述能力特徵典型使用案例
Level 0 提示助手 單輪問答，無記憶，無工具呼叫 基本 Q&A、內容生成
Level 1 工具呼叫 可使用 1-3 個外部工具，單次任務 搜尋輔助、計算器、API 查詢
Level 2 工作流程自動化 多步驟任務、條件判斷、跨系統操作 報告生成、資料分析流程、IT 工單處理
Level 3 多智能體協作 多個 agents 協同、角色分工、複雜協調 軟體開發自動化、研究綜合、供應鏈優化
Level 4 完全自主 目標驅動、自主規劃、最小人類干預 自主研究、業務決策自動化（仍在發展中）
⚠️四大核心風險

🎯 提示注入攻擊（Prompt Injection）

攻擊者在 agent 輸入中嵌入惡意指令，覆蓋原始目標或竊取敏感資料。多模態模型進一步擴大攻擊面，需要輸入驗證、沙盒隔離與行動審計。

🔗 錯誤累積（Compounding Errors）

在長序列任務中，早期的小錯誤可能在後續步驟中放大，導致最終結果嚴重偏離目標。需要設置中間檢查點與 human-in-the-loop 機制。

⚖️ 偏見與歧視風險（Bias Risk）

訓練資料中的偏見透過 agent 的自主決策被放大，特別危險於招募、信貸審核等高影響決策。需要代表性訓練資料與定期偏見審計。

🔒 過度授權風險（Over-Privilege）

若 agent 被授予超出任務所需的系統權限，一旦被入侵或行為異常，後果難以控制。必須嚴格執行最小權限原則（Minimum Privilege）。

✅最佳實踐清單
  • 最小權限設計：每個 agent 只獲得完成其特定任務的最小必要權限，並與現有 IAM 系統整合
  • 任務邊界清晰：在設計 agent 時明確定義哪些決策需要人類審核，建立不可逾越的紅線
  • Human-in-the-Loop：在不可逆操作（刪除、支付、發信）前強制加入人類確認節點
  • 行動記錄可追溯：所有 agent 行動必須有完整日誌，可用於除錯、審計與責任追蹤
  • 從小開始：先用最簡單的 Workflow（而非 Agent），只在必要時升級複雜度
  • 工具設計精良：工具名稱與描述極度清晰，每個工具只做一件事，回傳結構化 JSON
  • 持續監控：追蹤成功率、延遲、成本、用戶滿意度，設置異常警報
  • 對抗性測試：上線前執行 Red-teaming，模擬惡意輸入與邊緣情境
  • 代表性訓練資料：確保 fine-tuning 資料反映真實使用場景與多元用戶群
  • 定期偏見審計：每季度審查 agent 決策是否存在系統性偏見，特別是高影響決策
🚀三階段企業導入路線圖（McKinsey）
1

Phase 1：基礎建設（3-6 個月）

建立資料管道與 API 整合層；選擇核心 LLM 平台（OpenAI / Anthropic / Google）；設計 Agent Orchestration 框架；確立治理政策與安全準則

2

Phase 2：試點驗證（6-12 個月）

選擇 2-3 個高 ROI 使用案例進行 Pilot；建立評估指標體系（準確率、效率、成本節省）；培訓核心 AI 人才；從使用者反饋中快速迭代

3

Phase 3：規模化（12-24 個月）

建立 AI Center of Excellence（CoE）；將成功模式推廣至全組織；建立跨部門 agent 協作框架；持續學習與改善機制
