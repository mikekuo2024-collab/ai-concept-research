提示工程

Claude 提示框架

The Claude Prompt Framework

一套把模糊請求轉成精準成果的 9 步驟提示骨架，從「核心公式」到「對齊後再放手」。

  • 核心公式：［動作動詞］＋［具體交付物］＋［對象要完成什麼 X 達到 Y］。
  • 明確指定範圍、長度、格式，並要求回應前先完整讀完指定檔案。
  • 定義輸出語氣、對象反應與成功定義；用白話寫、把贅詞改成簡單字。
  • 執行前先用提問釐清、列出最重要的 3 條規則，再給 5 步執行計畫。
02
技能 Skills

Claude 技能速查表

The Claude Skills Cheatsheet

技能 (Skills) 把「程序知識」外部化、模組化，解決長流程的脈絡視窗超載問題。

  • 核心問題：關鍵指令被埋沒、注意力被無關內容稀釋、回應前後不一。
  • 運作方式：動態載入、漸進揭露、可組合、可攜帶、可在沙盒執行。
  • 技能就是一個資料夾：SKILL.md（指令）＋ resources ＋ scripts。
  • 最佳實務：描述要具體、一技能一流程、版本化；並稽核第三方技能避免提示注入。

資料基礎

AI 常用的資料庫

Databases used in AI

AI 系統依資料形態選用不同資料庫，六大類各有適用場景。

  • 圖形資料庫（Neo4j）：以實體與關聯儲存，適合社群/知識圖譜。
  • 關聯式（MySQL）：資料表結構；文件式（MongoDB）：彈性 JSON。
  • 時間序列（InfluxDB）：時間戳資料；鍵值（Redis）：高速鍵-值對。
  • 向量資料庫（Pinecone）：儲存高維度嵌入，做語意相似度搜尋。

代理 Agent

提示工程 vs 迴圈工程

Prompt Engineering vs Loop Engineering

兩種人機協作模式的根本差異：誰在推動每一步。

  • 提示工程：人寫提示→代理回應→人審查→人再寫提示…每一步由「人」推動。
  • 迴圈工程：人設定目標→觸發器啟動→代理執行→判斷是否達標，未達標就自我迭代。
  • 迴圈工程加入「達成目標？」的判斷節點，形成自動循環直到完成。
  • 結論：迴圈工程讓代理驅動整個流程，人只需設定目標與最後審查。

Cowork

Claude Cowork 一次性設定

One-time Setup for Claude Cowork

用「設定 → 建立 → 運作」三步驟，一次設定好就別再浪費時間與 token。

  • 設定：建立 Claude Cowork 資料夾與 about-me／outputs／templates 子資料夾。
  • 建立：請 Claude 訪談你，用答案寫成精簡（<2000 token）的個人脈絡檔。
  • 運作：在全域指令叫它先讀 ABOUT ME、不自動讀大檔，用講的並要它反問。
  • 小訣竅：指出它的錯別照單全收，每 20 則訊息開新階段，好成果存成範本。

學習法

用 Claude 學習任何主題的四種方法

Four ways to learn any topic with Claude

四種把 Claude 變成學習夥伴的方法，外加六個可收藏的學習提示。

  • 建立學習技能：把教學規則存成技能，之後該主題每次對話都沿用。
  • 開啟學習模式：讓它反問你、引導你自己得出答案，才記得住。
  • 建立學習專案：上傳 PDF/文章/筆記，讓它從你的來源回答。
  • 跑費曼迴圈：你向它解釋概念，它找出你的破綻，反覆到能無破綻解釋。

向量資料庫（9 步驟）

Vector Databases

向量資料庫如何儲存、索引與檢索嵌入，達成大規模快速相似度搜尋。

  • 嵌入→相似度搜尋→中繼資料：把資料轉成捕捉語意的高維向量並標註欄位。
  • 索引（HNSW/IVF/PQ）→檢索 Top-K→中繼資料過濾，縮小並精準化結果。
  • 儲存：向量、中繼資料、索引一起存，兼顧效率與檢索品質。
  • 在 RAG 中應用：切塊→嵌入→檢索相關脈絡→餵給 LLM 生成答案。

提示工程

寫出有效 Claude 提示的 9 個技巧

9 Tips to Write a Claude Prompt That Actually Works

九個立刻可用的提示撰寫技巧，從命名產出到先說目標。

  • 講出「產出物」而非任務；一開始就定好長度；把每個「不要」翻成「要」。
  • 用動作動詞開頭；選 Opus＋自適應思考逼出最大推理量。
  • 加「超越基本要求」一句；上傳你的口吻範例；有目的地控制工具（搜尋與否）。
  • 在任務前先說目標與對象——沒有目標的提示只是願望。

評估

使用者專業度分級

User Expertise Classifier

從提問用語與查證行為，把使用者的領域專業度分成五級。

  • 第1級新手：無專業術語、查證籠統、認不出 AI 的錯。
  • 第2–3級初/中級：開始用術語與領域具體性，但未深入方法論與取捨。
  • 第4級進階：展現領域知識、預判取捨、查證有針對性、能抓到錯誤。
  • 第5級專家：用精密行話、預判設計決策、精準針對弱點，會反過來糾正 AI。

迴圈工程 101

Loop Engineering 101

打造能自我工作、檢查、重試的代理迴圈，在你睡覺時也持續出貨。

  • 迴圈＝探索→規劃→執行→驗證→迭代的自我循環；任務需可重複、可客觀檢查。
  • 封閉迴圈（有界、有驗證）勝過開放迴圈（探索、易偏離）。
  • 六大基石：自動化、工作樹、技能、外掛工具、子代理、記憶。
  • 品質把關（編譯/測試/CI）＋自我學習迴圈（RULES.md）讓錯誤不再重犯。
