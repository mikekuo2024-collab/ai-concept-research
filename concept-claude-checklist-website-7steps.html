Claude 入門檢查清單

Claude Checklist

一張表把 Claude 從訂閱、設定到日常使用的 72 個要點分成 9 大面向，逐項打勾即可完成專業級配置（原作：Poornachandra Kongara）。

  • 基礎設定與連接器：從 Pro 訂閱、桌面版、Computer Use 到 Gmail／Drive／GitHub 等七大連接器，先把工作環境接好線；任務用不到的連接器記得關閉。
  • 情境資料夾與 Projects：用 ABOUT ME／OUTPUTS／TEMPLATES 三個子資料夾管理個人脈絡，每個重複性交付物建一個 Project 並上傳黃金範例、指令控制在 8 行以內。
  • 提示技巧：先講目標與成功標準、給一個好範例勝過空口描述、一次打包三個任務、話題改變就開新對話。
  • 技能與研究：用 skill-creator 建技能並以五種說法測試、描述加上「不要用於…」；研究時要求五次搜尋找缺口、標記來源矛盾、優先 2025–2026 來源。
  • 代幣經濟：先聊天規劃再到 Cowork 建置、PDF 與截圖先轉 Markdown、每 20 則訊息重開對話、簡單任務關閉延伸思考，把用量花在刀口上。

用 Claude Code 建立網站的 7 個步驟

7 Steps to Create a Website with Claude Code

一套「先規劃、再建置、專業部署」的建站工作流：目標 → 簡報 → 參考 → 規劃 → 迭代建置 → 部署 → 心法（原作：AIPlanetX.com）。

  • 步驟 1–3 定方向：先回答事業是什麼、網站給誰看、要訪客採取什麼行動；寫出含受眾、必要頁面、關鍵功能與主要 CTA 的詳細簡報；再提供參考網站的版面、字體、配色與動效——給方向，不要讓它瞎猜。
  • 步驟 4 先規劃：確定網站架構、頁面層級、導覽、響應式版面、元件清單與技術棧（Next.js、Tailwind CSS、TypeScript、PostgreSQL）——事前規劃勝過事後除錯。
  • 步驟 5–6 建置與部署：以小步迭代建置可重用元件、表單、身分驗證、API、SEO 與無障礙；部署前檢查行動裝置相容、頁面速度、失效連結、SEO 中繼資料與內容正確性，上線後收集回饋持續改進。
  • 步驟 7 最大的一課：明確目標＋詳細需求＋參考設計＋扎實架構＋迭代改進——Claude 負責寫程式，你負責提供願景。
