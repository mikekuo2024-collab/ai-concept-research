Step 1：從截圖開始

截下喜歡的網站直接丟進 Claude，不需用文字描述設計風格。

Step 2–3：樣板 + 魔法提示

下載 design.md 統一樣式；用「你是 CTO，我是 CEO」讓 Claude 主動訪談需求。

Step 4–5：繞過確認 + 逐頁推進

開啟 bypass 模式讓 Claude 自主執行；一次只交辦一個小任務。

Step 6–7：截圖修正 + Netlify 上線

描述問題或截圖 bug，5–6 輪修正後，Netlify 一鍵上線。

02 如何建立真正有效的 Claude Skills

Claude Skills 完整建立指南

原作：英文 AI 資訊圖 · 中文重製

核心原則

Skill 若無法被 Claude 識別，就等於失敗。描述要清楚、觸發詞要具體。

完美描述公式

任務 + 觸發詞組 + 範例句。讓 Claude 能在正確時機自動啟動。

一個 Skill = 一個任務

精準且輕量。SKILL.md 少於 500 行，重資訊移至輔助檔案。

失敗原因

描述模糊、一個 Skill 太多任務、沒有觸發關鍵字、沒有附範例。

03 聰明工作的 10 個 Claude 斜槓指令

10 個強大 Claude 指令速查表

原作：英文 AI 資訊圖 · 中文重製

/ghost + /brief

人性化寫作後精簡輸出，是內容創作者的基本組合。

/challenge + /roast

壓力測試想法、找出盲點，決策前必用。

/matrix + /steal

把選項整理成比較表，並逆向分析成功案例的背後原則。

黃金組合技

/ghost + /brief + /challenge：人性化→精簡→壓力測試，三步完成高品質輸出。
