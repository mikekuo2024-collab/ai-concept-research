如何真正用好 Fable 5：四層自我改進架構

Mythos-Tier Model. Sonnet-Tier Habits. — How to Actually Use Fable 5

頂級模型配上普通習慣，只會得到普通結果。這張圖解釋大多數人用 Fable 5 的方式（下提示→拿答案→明天從頭再來）與高手做法（執行→記錄→蒸餾→重複）的差異，核心是四層架構：基礎原語、協作編排、記憶、自我改進。

  • 大多數人止步於「基礎原語」層：只用模型與原始工具，沒有記憶、沒有學習、沒有複利。
  • 記憶層＝代理在每次工作階段結束前寫入、下次開始時讀取的狀態檔：已驗證的事實、蒸餾出的規則、有效做法與下一步。
  • 黃金法則：產出者 ≠ 驗證者——由獨立的驗證代理依評分標準檢查產出，而不是同一個代理自己批改自己的作業。
  • 自我學習（模型更新自身權重）目前沒有任何公開模型做得到；自我改進（模型不變、周邊系統持續變好）今天就能做。
  • 自我改進迴圈：執行 → 產出 → 獨立審查 → 學到教訓 → 下次執行（更好），讓明天的執行繼承今天磨利的系統。

Claude 自動化 · 實戰路線圖 02 / 02

打造你的第一個 Claude 自動化：五步路線圖

Build Your First Claude Automation — from repeat task to reusable system

從重複任務到可重用系統的五步實戰路線圖：找出重複、任務分流、繪製工作流、打造技能、選擇執行環境。第一個自動化要挑熟悉、小巧、容易檢查的每週例行工作。

  • ① 找出重複：寫下五項重複任務，圈選輸入條件清楚的那一項——別自動化混亂，要自動化「重複」。
  • ② 任務分流（四道閘門）：刪除低價值工作、把可重複步驟交給 Claude 自動化、把需要判斷的委派給人、其餘用 AI 加速。
  • ③ 繪製工作流：標出觸發（什麼啟動執行）、動作（接下來發生什麼）、審核（你在哪一步核准）——清楚的地圖讓提示變成系統。
  • ④ 打造技能：把地圖變成可重用的操作檔，內含目標（角色與步驟）、佐證（範例與驗收檢查）、工具（檔案、應用程式與指令）。
  • ⑤ 選擇執行環境：同一個技能可在聊天中手動執行、在 Cowork 排程執行、或移往雲端自主執行——同一技能，不同引擎。
